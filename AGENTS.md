# x AGENTS

## 适用范围

- 作用域：仓库根目录及未被子目录 `AGENTS.md` 覆盖的路径
- 目标：约束这个 Go 公共库集合的全局协同边界、执行优先级与接管机制

## 可改范围与禁区

- 可改：与当前任务直接相关的 Go 代码、测试、文档和根级控制面文件
- 禁区：未获授权的仓库外路径、生产环境操作、破坏性 Git 历史改写

## 停止条件

- 需求存在多种互斥实现，且无法从仓库事实直接裁决
- 变更会影响现有公开 API 或包边界，但当前任务没有明确迁移要求
- 缺少必要工具或环境，导致无法完成最小验证

## 人工接管条件

- 需要仓库外系统权限、敏感配置或发布动作
- 需要对公共 API 兼容性、版本策略或目录拆分做人工拍板
- 发现会影响现有使用方的行为变更，但无法在当前上下文确认范围

## 规则优先级

1. 仓库根 `AGENTS.md`（本文件）
2. 模块 `AGENTS.md`
3. 任务临时约束

## 新规则审校机制

1. 新规则必须说明目标、边界、风险和回滚方式
2. 生效前先检查是否与 root、模块或任务约束冲突
3. 冲突未消解前不得落地到仓库

## 协同约束

1. `README.md` 负责仓库概述与导航，`justfile` 负责稳定命令入口，不在两者之间混写
2. 新增子模块前必须先确认是否真的新增，以及是否需要单入口或多入口执行面
3. 变更公共包时优先保持最小影响面，不做与目标无关的顺手重构

## 全局编码倾向

### 实现风格

1. 不做与目标无关的功能扩展、顺手重构或批量风格清理。
2. 优先通过参数传递上下文，避免可变全局状态。
3. 除专有名称外，新增文档、注释与说明尽量使用中文。
4. 避免过度工程、过度防御、过早抽象与无必要注释。

### 依赖与环境

1. 不自行安装依赖、工具或系统组件。
2. 缺依赖导致阻塞时，明确记录为环境前置条件。

### 开发与测试

1. 非极小任务先拆分再执行。
2. 测试描述优先准确中文，直接表达业务语义、输入条件与预期。
3. 无法补测时必须说明缺口与原因。
