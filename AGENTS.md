# x AGENTS

## 适用范围

- 作用域：仓库根目录及未被子目录 `AGENTS.md` 覆盖的路径
- 目标：约束这个 Go 公共库集合的全局协同边界、执行优先级与接管机制

## 规则优先级

1. 仓库根 `AGENTS.md`
2. 子目录 `AGENTS.md`
3. 任务临时约束

## 变更边界

- 可改：与当前任务直接相关的仓库内代码、测试、文档和控制面文件
- 禁区：仓库外路径、生产环境操作、破坏性 Git 历史改写
- 要求：不做与任务无关的功能扩展、顺手重构或批量风格清理

## 停止条件

- 需求存在多种互斥实现，且无法从仓库事实直接裁决
- 变更会影响现有公开 API、包边界或兼容性，但当前任务没有明确迁移要求
- 缺少必要工具或环境，导致无法完成最小验证

## 人工接管条件

- 需要仓库外系统权限、敏感配置或发布动作
- 需要对公共 API 兼容性、版本策略或目录拆分做人工拍板
- 发现会影响现有使用方的行为变更，但无法在当前上下文确认范围

## 文件边界

- `AGENTS.md`：只承载协作约束、边界、停止条件与接管条件
- `README.md`：只承载仓库概述、职责边界与索引
- `justfile`：只承载稳定执行入口，不混写规则和背景说明

## 基本工程约束

- 不因目录已存在而默认新增模块、执行入口或注册关系
- 变更公共包时优先保持最小影响面，不破坏长期分层一致性
- 优先通过参数传递上下文，避免可变全局状态
- 新增文档、注释、测试描述、错误和日志说明尽量直接、准确、可验证；除专有名称外优先使用中文
- 无法补测或无法完成最小验证时，必须明确说明缺口与原因
- 若发现项目结构、模块分层、入口组织或控制面职责不符合预期，必须先指出问题，再给出可执行的整理建议

## 规则启用机制

- 只有落盘到 `AGENTS.md` 的规则才生效
- 子目录 `AGENTS.md` 只重写确有必要的局部约束
- 未被下级重写的规则继续继承上级；冲突约束由更近的 `AGENTS.md` 覆盖
